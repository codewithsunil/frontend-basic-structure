# 🚀 Project Name A modular **HTML + JavaScript boilerplate** with clean architecture for building scalable web applications (CRUD, dashboards, SaaS, etc.). --- ## 📂 Project Structure ``` project/ ├── index.html ├── .env.js ├── assets/ │ ├── css/ │ ├── js/ │ │ ├── config/ # Environment & app config │ │ ├── services/ # API layer │ │ ├── modules/ # Feature-based modules │ │ ├── utils/ # Helper functions │ │ └── app/ # Entry files (per page) ``` --- ## ⚙️ Setup 1. Clone project 2. Open in browser OR use local server ``` npx serve . ``` --- ## 🔑 Environment Config Edit `.env.js`: ```js window.__ENV__ = { API_BASE_URL: "http://localhost:8000/api", API_KEY: "your-api-key", APP_ENV: "development", APP_VERSION: "1.0.0", }; ``` --- ## 🧠 Architecture ### 🔹 Config - API base URL - Headers - Environment variables ### 🔹 Services - All API calls - Example: `postService.js` ### 🔹 Modules - Feature-based logic (UI + behavior) - Example: `PostModule` ### 🔹 Utils - Helper functions (storage, validation, etc.) --- ## 📄 Page-based Entry Each page loads its own JS: ```html ``` --- ## 🔌 Example Usage ```js import { getPosts } from "../services/postService.js"; const posts = await getPosts(); console.log(posts); ``` --- ## 🏗️ Features - ✅ Modular JS architecture - ✅ Service-based API layer - ✅ Environment config support - ✅ Page-based entry system - ✅ Scalable folder structure --- ## ❗ Best Practices - Do NOT mix UI and API logic - Keep services clean (no DOM) - Use modules for features - Avoid global variables --- ## 🚀 Future Improvements - Loader system - Toast notifications - Validation module - Auth system (OTP, login) --- ## 👨‍💻 Author
